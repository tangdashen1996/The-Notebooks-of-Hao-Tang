# Python 导入路径小结

## 基本概念

Python 导入包路径主要分为两类：

- **绝对导入**：从项目或环境中可识别的顶层包开始写完整路径。
- **相对导入**：使用 `.` 或 `..` 表示当前包或上级包中的模块。

标准库导入不需要区分绝对导入和相对导入；这个问题主要出现在项目内部包之间的导入。

## 当前工作目录与文件路径

执行 `.py` 文件时的当前工作目录会影响：

- 通过 `os`、`pathlib` 等模块处理的相对路径。
- 程序运行时读取或写入文件的位置。

但它不等同于模块导入路径。模块导入还会受到 `sys.path`、包结构、执行方式等因素影响。

## `python xxx.py` 与 `python -m package.module`

在 `sys.path` 使用默认值的情况下：

- 使用 `python xxx.py` 直接执行文件时，该文件会被当作顶层脚本，包内相对导入可能失败。
- 使用 `python -m package.module` 执行模块时，Python 会按包结构识别模块，更适合运行包内代码。

## 绝对导入与相对导入的经验规则

- 绝对导入适合清晰表达模块来源，通常更推荐。
- 相对导入适合包内部模块之间的短路径引用。
- 如果某个模块需要相对导入，尽量不要用 `python xxx.py` 直接运行它。
- 项目复杂后，建议从项目根目录使用 `python -m package.module` 运行模块。

## 记忆要点

- 文件系统路径和模块导入路径是两个概念。
- 直接运行脚本时，相对导入最容易出问题。
- 包内模块建议通过 `python -m` 方式运行。
