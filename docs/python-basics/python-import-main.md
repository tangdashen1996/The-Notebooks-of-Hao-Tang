# Python 模块导入与 `if __name__ == "__main__"`

## 问题背景

如果一个模块既定义函数，又在顶层写了可执行代码，那么其他文件导入该模块时，顶层代码也会被执行。

这通常不是我们想要的结果：导入模块时应该只加载函数、类、常量等定义，而不是自动运行脚本逻辑。

## 推荐写法

把直接运行脚本时才需要执行的代码放进 `main()`，再使用 `if __name__ == "__main__"` 保护起来。

```python
def say_hello(name):
    print(f"Hello, {name}!")


def main():
    say_hello("Hao Tang")


if __name__ == "__main__":
    main()
```

## 为什么这样写有效

- 当文件被直接运行时，`__name__` 的值是 `"__main__"`。
- 当文件被其他模块导入时，`__name__` 的值通常是模块名。

因此，`if __name__ == "__main__"` 下面的代码只会在直接运行该文件时执行。

## 记忆要点

- 可复用逻辑放在函数或类中。
- 脚本入口放在 `main()` 中。
- 入口调用使用 `if __name__ == "__main__"` 保护。
